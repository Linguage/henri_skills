# 第一阶段：根目录文件分类

将 Downloads 根目录散落文件按扩展名分类到对应文件夹。

## 文件分类规则

#### 书籍 (.pdf, .epub, .mobi)
移动到 `书籍/`

#### 文档 (.pptx, .xlsx, .docx, .txt, .html, .json 配置文件, .md)
移动到 `文档/`

#### 图片 (.png, .jpg, .jpeg, .gif, .svg, .webp, .bmp, .heic, .tiff)
使用三层分类系统（详见「第四阶段」），**必须通过 `classify_images.py` 脚本分类**：
```bash
conda run -n henri_env python .claude/skills/organize-downloads/scripts/classify_images.py --scan-root
```
- 先运行报告模式，确认分类结果后再加 `--move` 执行移动
- Gemini nano 图片 (Gemini_Generated_Image_*) → `图片/AI生成图片/Gemini/` 或 `图片/gemini-nano/`
- 其他所有图片按三层系统分类到对应子目录

#### 视频 (.mp4, .mov, .avi)
移动到 `视频/`

#### 安装包 (.dmg, .pkg, .zip 安装文件)
移动到 `安装包/`

#### 数据文件 (.csv, .json 数据文件)
移动到 `数据文件/`

#### 音频 (.m4a, .mp3, .wav)
移动到 `音频/`

#### 系统文件 (.DS_Store, .localized)
移动到 `待删除/`，等待用户确认

## 敏感文档识别

在整理文档前，必须识别可能包含个人信息的文件，并提示用户单独存档。

**敏感文档特征：**
- **姓名标识**：文件名包含个人姓名拼音（如 `姓氏缩写-`, `全拼-` 等模式）
- **学校/机构**：大学、学院、公司名称（如 `XX大学`, `XX学院` 等）
- **证件类型**：登记表、申请表、证明、证书、证件、成绩单、简历
- **法院文件**：`gov.uscourts`, `court`, `case` 等司法文件
- **身份证件**：身份证、护照、驾驶证、社保卡
- **财务信息**：工资单、银行流水、税单、发票

**敏感文档处理：**
1. 单独列出，标注 `敏感文档`
2. 询问用户是否需要专门存档到 `个人档案/` 或其他指定位置
3. 用户确认后再执行移动

## 工作流程

1. 扫描 Downloads 根目录的所有文件
2. 根据文件扩展名和命名规则分类
3. **检测敏感文档**，单独列出并询问用户存档位置
4. 列出建议的移动操作，**必须等待用户确认**
5. 用户确认后，执行 mv 命令移动文件
6. 系统文件移动到 `待删除/` 文件夹，不直接删除
